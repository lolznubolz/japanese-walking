# iOS-сборка «Японская ходьба» — что готово и что нужно от Apple

## Что я сделал

В проекте не было iOS вообще. Добавил три файла, повторяющие вашу Android-логику
(папка `ios/` не хранится в репозитории, а создаётся в CI на лету):

| Файл | Куда положить в репозитории | Назначение |
|---|---|---|
| `build-ios.yml` | `.github/workflows/build-ios.yml` | Сборка iOS на macOS-раннере: `flutter create --platforms=ios` → звук → патч Info.plist → `flutter build ios`. |
| `patch_ios.py` | `japanese_walking/tools/patch_ios.py` | Правит Info.plist: разрешения Bluetooth (для пульса), фоновые режимы (метроном/пульс при гашении экрана), русское имя на иконке. |

После добавления в `main` сборка запустится сама и **проверит, что приложение компилируется под iPhone** (без подписи). Это и есть готовность iOS-кода к запуску.

## Чего пока нельзя без аккаунта Apple

iOS физически не даёт поставить приложение на iPhone или в TestFlight **без подписи**, а подпись возможна только с **Apple Developer Program ($99/год)**. Поэтому сейчас сборка идёт в режиме `--no-codesign`: код собирается, но установочный `.ipa` не создаётся.

## Когда купите аккаунт — порядок действий

1. **Apple Developer** — `developer.apple.com/programs/enroll`, $99/год, верификация 1–2 дня.
2. **App Store Connect** → создать приложение, Bundle ID **`app.walkbeat.japaneseWalking`** (его генерирует `flutter create`; для iOS он в camelCase, это нормально).
3. **App Store Connect API-ключ** (Users and Access → Integrations) — даст автоматическую подпись и заливку в TestFlight без Mac под рукой.
4. Добавить в GitHub секреты репозитория: `APP_STORE_CONNECT_KEY_ID`, `APP_STORE_CONNECT_ISSUER_ID`, `APP_STORE_CONNECT_KEY_P8`.
5. Я раскомментирую в `build-ios.yml` блок подписи → сборка станет подписанным `.ipa` и сама поедет в **TestFlight** → оттуда на ревью в App Store (1–3 дня).

Тексты для карточки (название, описание, ключевые слова) уже есть в маркетинг-паке — переиспользуем.
